## 표준과 프로토콜의 세계, 인터넷

#### 인터넷

느슨하고 체계가 없으며 혼란스럽고 임시적인 네트워크 모음이며, 그 위의 컴퓨터가 서로 통신하는 방법을 규정한다.



#### 멀리 떨어진 네트워크의 연결은 어떻게 이루어질까?

네트워크와 컴퓨터를 식별하기 위해서 이름과 주소가 필요한데(주소록처럼) 그 주소를 찾은 다음

👉 직접 연결되지 않은 네트워크 사이의 경로를 찾을 수 있어야 한다. 



#### 프로토콜

'약속!' - 소통하기 위한 일련의 규칙

모든 네트워크에서 인터넷에서 오는 데이터를 어떤 형식으로 구성할지, 누가 먼저 말할 것이고 어떤 응답이 있는지, 오류 처리 등에 대한 합의는 프로토콜로 처리된다.



## 인터넷이 가능한 매커니즘

#### 아파넷 - 인터넷의 시작

- 지리적으로 멀리 떨어진 위치에 컴퓨터를 연결하는 네트워크를 구축하려고 시도

- 네트워크 구성 요소 중 하나에 이상이 생기더라도 견고하게 작동하고, 문제가 발생한 부분을 우회해서 트래픽을 라우팅하도록 설계

- 아파넷이 상업적 영역으로 퍼져 나가며 인터넷으로 불리게 됐다



#### 인터넷

수백만개의 독립적인 네트워크로  구성

가까이 있는 컴퓨터끼리는 근거리 통신망 - 무선 이더넷

게이트웨이 또는 라우터를 통해 다른 네트워크에 연결 : 무선 이더넷

근거리 네트워크들이 게이트웨이, 라우터를 통해 다른 네트워크에 연결

게이트워이 라우터

**게이트웨이** : 한 네트워크에서 다음 네트워크로 정보 패킷 라우팅하는데 전문화된 컴퓨터

게이트웨이는 라우팅 정보를 서로 교환하여 연결 상태를 파악한다.



#### 패킷

형식이 저장된 일련의 바이트

주소 정보를 포함한다 - 어디서 오고 어디로 가는지..

페이로드 : 패킷이 전달하는 정보



인터넷에서 데이터는 IP패킷으로 전달된다

IP패킷은 모두 같은 형식이며, 크기가 클 경우 쪼개서 하나 이상의 물리적 패킷에 실려 전송된다.

IP패킷은 여러 게이트웨이를 통과하며, 게이트웨이를 통과할 때 더 가까이 있는 게이트웨이로 패킷을 보낸다.



#### IP패킷의 작동 매커니즘

1. 식별하기 위한 주소 - 4바이트인 IPv4, 16바이트인 IPv6(16진수)

   IPv4는 거의 소진되었으므로 IPv6으로의 전황이 가속화된다.

2. 이름 :

   - 도메인 네임(www.naver.com)
   - 도메인 네임 시스템

3. 라우팅

   - 목적지까지의 경로 찾기 - 게이트웨이가 제공
   - 라우팅 정보를 계속 교환하고, 가까운 게이트웨이로 보내준다.

4. 프로토콜

   - 다른 컴퓨터로 정보가 복사되도록 구성 요소가 어떻게 상호 운용되는지 정확하고 자세하게 설명하는 규칙과 절차

#### TCP

- Transmission Control Protocol - 전송 제어 프로토콜
- 출발지에서 목적지까지 임의 길이의 바이트 시퀀스를 전송하기 위한 안정적인 매커니즘

## 쿠키를 삭제하시겠습니까?

HTTPS : 무상태 프로토콜

무상태 프로토콜 : 서버가 클라이언트 요청에 대해 아무것도 기억하지 않아도 된다.

기억하지 않으면 데이터 교환 기록을 모두 버리게 되고, 서버가 방문자의 정보에 대해 접근할 경로가 없게 된다😅

이를 해결하기 위해 넷스케이프가 `쿠키`를 발명해낸다.



#### 쿠키

서버가 브라우저에 웹페이지를 보낼 때, 브라우저가 저장하게 되어 있는 추가 텍스트 덩어리를 포함한다.

각각의 덩러리를 쿠키(각각 최대 4000바이트)라고 한다.

이후에 브라우저가 같은 서버에 요청을 보낼 때, 브라우저는 그 쿠키를 도로 전송한다 (서버가 클라리언트에 고유한 식별 번호를 할당하고 번호를 쿠키에 포함하는 것)

로그인 상태, 장바구니 내용, 사용자 설정



## 어도비 플래시는 왜 퇴출됐을까?

#### 액티브 콘텐츠

브라우저를 통해 웹에서 코드를 다운로드하여 실행하는 콘텐츠

액티브 컨텐츠는 인터넷 설문 조사, JavaScript 애플리케이션, 주식 시세 표시기, 애니메이션 이미지, ActiveX 애플리케이션, 액션 아이템, 스트리밍 비디오 및 오디오, 날씨 맵, 내장 된 객체 등과 같은 프로그램을 포함하는 대화 형 또는 동적 웹 사이트 유형

브라우저 내에서 자바  인터프리터를 포함해 연산 등을 수행시키다가 마이크로소프트가 견제를 시작 ->

/ 자바는 브라우저를 확장하는 방법으로는 인기를 얻지 못했다

넷스케이프가 자사 브라우저에 사용할 새로운 언어인 자바스크립트를 만들었다.



### 자바스크립트/자바

#### 공통점

둘 다 가상 머신을 구현해 사용



#### 차이점

**자바**

- 자바소스 코드는 만들어진 곳에서 생성되고, 그 결과로 오브젝트 코드가 생성되며 이가 브라우져로 보내져 해석된다. 

**자바스크립트**

- 소스 코드가 브라우저로 보내지고 거기서 컴파일된다

- 그래픽 효과 제공, 폼 정보 유효성 검사, 팝업 창 띄우기

- 정교한 추적과 모니터링



#### 플러그인

필요에 따라 브라우저에 동적으로 로드되는 프로그램

브라우저가 직접 처리할 수 없는 포맷의 컨텐츠가 있을 시 사용된다.

👉 브라우저와 긴밀하게 협력하여 컴퓨터에서 실행될 새 프로그램을 다운로드
